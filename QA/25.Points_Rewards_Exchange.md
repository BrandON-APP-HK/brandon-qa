# 25. Points Rewards Exchange

## Scope

Rewards list states and reward exchange bottom sheet.

## Preconditions

- User is logged in.
- Reward data includes available, insufficient-points, sold-out, and address-required examples when possible.

## Scenario 25.1: Reward card states

- Open My Points active tab.
- [ ] Reward cards should show image, name, price, and points label.
- Observe insufficient-points reward.
- [ ] It should appear unavailable and not open exchange sheet.
- Observe sold-out reward.
- [ ] Sold-out state/overlay should show and exchange should not open.
- Observe address-required reward with incomplete profile.
- [ ] It should be unavailable until address is complete.

## Scenario 25.2: Open and cancel exchange sheet

- Tap an available reward.
- [ ] Exchange bottom sheet should open.
- [ ] Sheet should show user points, reward image, name, description, price, **Exchange**, and **Cancel**.
- Tap Cancel or dismiss sheet.
- [ ] Sheet should close.

## Scenario 25.3: Successful exchange

- Tap an available reward.
- Tap **Exchange**.
- [ ] Exchange API should be called.
- [ ] Sheet should close on success.
- [ ] My Points should switch to History tab.
- [ ] Success toast should appear.

## Scenario 25.4: Exchange failure

- Open exchange sheet for available reward.
- Simulate API failure.
- Tap **Exchange**.
- [ ] Failure toast should appear.
- [ ] User should be able to retry or cancel.
