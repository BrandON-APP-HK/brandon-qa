# 24. Points Active History

## Scope

My Points active points, active-points info, history tab, refresh, and navigation.

## Preconditions

- User is logged in.
- User has point balance and transactions.

## Scenario 24.1: Active points tab

- Open drawer > **My Points**.
- Active Points tab should be selected by default.
- Current points balance should display.
- Pull to refresh.
- Points and rewards should refetch.
- Tap info icon.
- Active points info modal/page should open.
- Close it.
- My Points should remain accessible.

## Scenario 24.2: Earn more points

- Open My Points.
- Tap **Earn more points**.
- App should navigate to Home.

## Scenario 24.3: Missing address prompt

- Use account without full address.
- Ensure at least one reward requires full address.
- Open My Points.
- Address completion prompt should appear.
- Tap it.
- App should navigate to Account address section.

## Scenario 24.4: History tab

- Open My Points.
- Tap **History** tab.
- Transactions should group by date.
- Newest transactions should appear first.
- Incoming amounts should show `+`.
- Outgoing amounts should show `-`.
- Reason labels should include check-in, pending, purchase, publish deal, or other as applicable.
- Pull to refresh.
- Transactions should refetch.
