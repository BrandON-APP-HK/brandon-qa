# 17. Check In Eligibility States

## Scope

Implemented non-QR check-in button states and info modal behavior.

## Preconditions

- Traffic/check-in campaign exists with store geo point, opening hours, check-in range, available check-in pool/capacity, and API support.
- Device location can be changed.
- QR/scanner check-in is out of scope.

## Scenario 17.1: Logged-out state

- Start logged out.
- Open a check-in eligible card or brand page. Note : for all logged-out users, the "check-in eligible" button reads "earn points".
- Check-in button should show earn-points/check-in copy.
- Button should use primary color for auth-required state.
- Tap check-in.
- Login bottom sheet should open.
- Tap info icon.
- Check-in info modal should open (depending on state, displays info about status of the button)

## Scenario 17.2: Location denied state

- Log in.
- Deny foreground location permission.
- Open check-in eligible item.
- Button should show earn-points copy.
- Tap check-in.
- App should request location permission when it can ask again.
- If permission is blocked, continue.
- App should show location permission error/toast behavior and route safely.

## Scenario 17.3: Store closed

- Use a store whose opening hours do not include the current local time.
- Open check-in eligible item while logged in.
- Button should use inactive/dark styling.
- Tap check-in or info.
- Check-in info modal should open
- Modal should indicate opening-hours/closed-store information. Improvement : display the opening hours

## Scenario 17.4: Not close enough

- Log in and allow location.
- Set device location outside campaign `checkInRange`.
- Open check-in eligible item.
- Button should use inactive/dark styling.
- Tap check-in.
- Info modal should open instead of awarding points.
- Modal should indicate check-in is not available/not close enough.

## Scenario 17.5: Already checked or cooldown

- Use an account/store where the deal is already in check-in history, or complete one check-in.
- Return to same eligible item.
- Button should show checked or waiting/cooldown styling.
- Tap check-in.
- Info modal should open.
- Points should not be awarded again.
- Wait for cooldown if testing recent check-in.
- State should update after cooldown according to app behavior.

## Scenario 17.6: Ready state

- Log in and allow location.
- Set device location inside campaign `checkInRange`.
- Ensure check-in campaign is active.
- Ensure store is open.
- Ensure campaign still has available check-in pool/capacity.
- Ensure user is not already checked in or cooling down.
- Open check-in eligible item.
- Button should show check-in copy.
- Button should use primary color.
- Info icon should still open info modal.
