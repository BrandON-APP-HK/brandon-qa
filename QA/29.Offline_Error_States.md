# 29. Offline Error States

## Scope

Global offline screen and source-backed recovery/error behavior.

## Preconditions

- Network can be disabled/enabled.
- App is installed and runnable.

## Scenario 29.1: Global offline screen

- Launch app online.
- Disable network connectivity.
- Wait for connection listener/check.
- Normal navigation should be replaced by No Connection screen.
- No-connection text/icon should be visible.
- Refresh button should be visible.
- Tap Refresh while still offline.
- App should remain offline or reload/check according to platform behavior.
- Restore network and tap Refresh or wait.
- App should return to normal root navigation.

## Scenario 29.2: Add Deal submit recovery

- Open Add Deal and complete required fields.
- Disable network.
- Submit.
- Submit loading should appear.
- Failure recovery card should appear.
- Restore network and tap Retry.
- Submit should retry with same data.
- Success should navigate Home with toast.

## Scenario 29.3: Add Deal store search recovery

- Open Add Deal with location available.
- Disable network.
- Type store search query longer than two characters.
- Search error should display.
- Restore network and retry.
- Store search should run again and show results when available.

## Scenario 29.4: Profile and account errors

- Open Account and change avatar while network is unreliable.
- Avatar upload failure should show failure toast.
- Submit wrong old password in Change Password.
- Invalid password error should show.
- Trigger delete-account failure requiring re-login.
- Error toast should ask user to log in again and route safely.

## Scenario 29.5: Reward exchange failure

- Open My Points and an available reward exchange sheet.
- Disable network or force API failure.
- Tap **Exchange**.
- Failure toast should show.
- User should be able to cancel or retry after restoring network.
