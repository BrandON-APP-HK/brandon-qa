# 9. Home Feed Browse Filter Sort

## Scope

Home feed browsing, category filtering, sorting, distance sort, refresh, and pagination.

## Preconditions

- API has multiple social deals, promo blocks, categories, and enough items for pagination.
- Test as guest and logged-in user.

## Scenario 9.1: Guest browses home feed

- Start logged out.
- Open **Home**.
- Initial Home feed should use backend-default ordering.
- Date, rating, or distance sorting should apply only after user selection.
- Feed should load social deal cards and promotion blocks.
- Category pills should be visible when categories are returned.
- Date and rating sorting should be available.
- Distance sort should not be available for guest users. It should remain as a visible option, though not clickable.
- Scroll the feed.
- Cards should remain readable and images should load or show placeholders.

## Scenario 9.2: Category filter

- Log in.
- Open **Home**.
- Tap a non-default category pill.
- Selected category should be highlighted.
- Feed should reload using that category.
- Tap another category.
- Active category should update.
- Category bar should keep the selected item visible.
- Category bar should scroll until selected category is displayed on the far left of the bar, unless the bar is scrolled all the way to the right and displays the visible category among the last ones of the scroll.

## Scenario 9.3: Date and rating sorting

- Open Home as logged-in user.
- Open sort picker.
- Select **Rating**.
- Sort label/state should update to Rating. best ratings on top.
- Feed should reload.
- Open sort picker again.
- Select **Date**.
- Sort label/state should update to Date. Newest on top.
- Feed should reload.

## Scenario 9.4: Distance sorting

- Log in.
- Reset location permission so the app can request it.
- Open Home and choose **Distance** sort.
- Grant location permission.
- Distance sort should be available for logged-in user. Shortest distance on top.
- App should request/use current location.
- Feed should wait for location and then display distance-sorted results.
- Repeat with location denied.
- Active sort should not switch to Distance when permission is denied.

## Scenario 9.5: Refresh and pagination

- Open Home with enough feed items.
- Scroll more than one screen down.
- Pull to refresh.
- Refresh spinner should appear.
- Current category/sort filters should remain applied.
- Feed should refetch.
- Scroll to the bottom.
- Next page should load when more pages exist.
- Duplicate pagination calls should not visibly duplicate items.
