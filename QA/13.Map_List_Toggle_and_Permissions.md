# 13. Map List Toggle and Permissions

## Scope

Map access, location permission, default region, find-store map/list switching, and close behavior.

## Preconditions

- User account exists.
- At least one brand campaign has store locations.
- Location permission can be reset.

## Scenario 13.1: Guest map access guarded

- Start logged out.
- Tap Home header map/list toggle or a deal address/map action.
- Login bottom sheet should open.
- Map should not open for the guest through guarded actions.

## Scenario 13.2: Map with location allowed

- Log in.
- Reset location permission and open **Map** from Home.
- Grant foreground location.
- Map should open.
- User location marker should appear when location is available.
- Recenter button should be visible.
- Tap recenter.
- Camera should move to user location.
- Category pills should be visible on general map.
- **REDO SEARCH** should be visible on general map.

## Scenario 13.3: Map with location denied

- Log in.
- Deny foreground location.
- Map does not open, with popup asking for mocation permission every time user clicks on a map icon, until it is given.

## Scenario 13.4: Find-store map to list and back

- Log in.
- Open a brand campaign with **Find a store**.
- Tap **Find a store**.
- Find-store map should open.
- Header should show **Find a store**.
- General category pills should not show in find-store mode.
- Tap list icon.
- **Find Store List** should open.
- Store list should show campaign stores when available.
- Tap map icon.
- App should return to map with the same brand campaign context.

## Scenario 13.5: Map close behavior

- Open map from Home.
- Tap map/list toggle back to list.
- App should return to **Home**.
- Open map from Search.
- Tap map/list toggle.
- App should return to **Search**.
- Open find-store from Brand Page and close.
- App should return to the brand page context.
- Note : list/map are different views of the same data, so users can toggle between the 2 seamlessly
