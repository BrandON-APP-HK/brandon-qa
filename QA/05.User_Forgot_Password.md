# 5. User Forgot Password

## Scope

Password reset request and resend flow.

## Preconditions

- Tester is logged out.
- Existing email/password account is available.

## Scenario 5.1: Open forgot password from password screen

- Enter an existing email in email login.
- Continue to password screen.
- Tap **Forgot password**.
- App should route to `/forgot-password`.
- Email field should show the selected email where supported.

## Scenario 5.2: Submit reset email

- Open `/forgot-password`.
- Enter invalid email.
- Invalid email should be rejected.
- Enter an existing valid email.
- Tap **Continue**.
- Password reset email request should be sent.
- App should route to `/send-email`.
- Confirmation screen should show the submitted email.
- **Resend link** and **Back to login page** should be available.

## Scenario 5.3: Resend reset link

- On `/send-email`, tap **Resend link**.
- Another reset email request should be sent.
- Confirmation screen should remain visible.
- App should not crash.

## Scenario 5.4: Back to login

- On `/send-email`, tap **Back to login page**.
- App should route to `/login`.
