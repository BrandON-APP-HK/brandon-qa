# 10. Home Deal Actions Like Share Report

## Scope

Deal and promo card actions from Home: details, map/auth gating, like, share, report/flag.

## Preconditions

- Feed contains at least one social deal and one promo block.
- Test as guest and logged-in user.

## Scenario 10.1: Guest protected actions

- Start logged out.
- Open Home.
- Tap a social deal heart.
- Login bottom sheet should open.
- Dismiss and tap report/flag.
- Login bottom sheet should open.
- Dismiss and tap address/map action.
- Login bottom sheet should open.
- Tap a social deal image.
- Deal image should open without requiring login.
- Tap a promo block image.
- Deal image should open without requiring login. Button "brand page" should be visible and clickable so user can access brand page.

## Scenario 10.2: Like and unlike

- Log in.
- Open Home.
- Tap heart on an unliked social deal.
- Deal should immediately appear liked.
- Tap the same heart again.
- Deal should appear unliked.
- Tap repeatedly during loading.
- UI should not create duplicate or unstable liked state.

## Scenario 10.3: Report a deal

- Log in.
- Open Home.
- Tap report/flag on an unflagged social deal.
- Report modal should open.
- Reasons should include finished promotion, store not found, and spam/fake promotion.
- Tap **OK** without selecting a reason.
- Modal should remain open and no report should submit.
- Select a reason and tap **OK**.
- Modal should close after submit.
- Deal should be treated as flagged.
- After the **2nd** valid report, the deal should automatically hide from user-facing lists.
- In JetAdmin, admin should be able to restore the hidden deal.
- In JetAdmin, admin should be able to delete/remove the reported deal.

## Scenario 10.4: Share a deal or promotion

- Open Home.
- Tap share on promo block (note : social deals do not have a share icon).
- Native share sheet should open.
- Cancel share.
- App should not crash.
- User should remain on the same feed.
- tap share again
- native share sheet should open
- user choose emai/whatsapp.., then be redirected to the corresponding service showing pre-written text and image to share.
- when sharing is finished, user is back to the app page to continue surfing 

## Scenario 10.5: Detail and map navigation

- Log in.
- Open Home.
- Tap a social deal image.
- **Deal Image** should open (note "deal details"). Image can be zoomed.
- Close details.
- tap a Promo bloc image
-  Deal image should open. Button "brand page" should be visible and clickable so user can access brand page.
- 
- Tap the deal address/map action.
- **Map** should open with the selected deal context.
- Return to Home and tap a promo block body/logo.
- Brand campaign page or promo detail should open when campaign data exists.
