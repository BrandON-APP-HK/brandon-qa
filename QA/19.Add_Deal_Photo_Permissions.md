# 19. Add Deal Photo Permissions

## Scope

Add Deal camera/gallery permission handling and photo preview.

## Preconditions

- User can open Add Deal.
- Camera and Photos permissions can be reset.

## Scenario 19.1: Camera permission granted

- Log in and open **Add Deal**.
- Tap **Take Photo**.
- Grant camera permission.
- Capture and confirm a photo.
- Native camera capture controls should be used.
- Selected image should appear fully visible in the deal preview using fit/contain behavior.
- Final preview should become available after a photo exists.
- Selected image should remain fully visible in the final preview using fit/contain behavior.

## Scenario 19.2: Camera denied or blocked

- Reset/deny camera permission.
- Open Add Deal and tap **Take Photo**.
- Deny permission.
- Recovery card should show camera permission message.
- If permission can be requested again, Retry should be available.
- Block camera permission in settings and retry.
- Open Settings action should be available.
- Choose from Gallery fallback should be available.

## Scenario 19.3: Gallery permission states

- Open Add Deal and tap **Choose from gallery**.
- Test full access.
- Image can be selected and previewed.
- Test limited access on iOS.
- Limited access information/recovery should be shown and picker should still open.
- Test denied permission.
- Retry recovery should be shown.
- Test blocked permission.
- Open Settings recovery should be shown.

## Scenario 19.4: Picker cancellation and errors

- Open camera or gallery picker.
- Cancel without selecting media.
- No photo should be added.
- App should remain on Add Deal.
- Force camera/gallery processing error if possible.
- Recovery card should show retry/fallback actions.
