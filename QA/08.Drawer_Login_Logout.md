# 8. Drawer Login Logout

## Scope

Drawer footer login/logout behavior and post-logout state.

## Preconditions

- Tester can log in with a valid account.

## Scenario 8.1: Drawer login entry

- Start logged out.
- Open drawer.
- [ ] Footer should show **Login**.
- Tap **Login**.
- [ ] Login bottom sheet should open.
- [ ] Google, Email, and iOS Apple options should be visible as applicable.

## Scenario 8.2: Drawer logged-in state

- Log in successfully.
- Open drawer.
- [ ] Drawer should show logged-in welcome/header content.
- [ ] Auth-only drawer items should be visible.
- [ ] Footer should show **Logout**.
- [ ] Version text should be visible.

## Scenario 8.3: Logout

- While logged in, open drawer.
- Tap **Logout**.
- [ ] App should dismiss to `/home`.
- [ ] User should be signed out.
- [ ] Drawer footer should change to **Login**.
- [ ] Auth-only drawer items should become hidden.
- Tap **Me**.
- [ ] Login bottom sheet should open.
