# 4. User Registration Email

## Scope

Email registration through the email-first auth flow.

## Preconditions

- Tester is logged out.
- A new unused email inbox is available.

## Scenario 4.1: New email routes to registration

- Open login bottom sheet.
- Tap **Continue with Email**.
- Enter an unused valid email.
- Tap **Continue**.
- [ ] App should route to `/registration`.
- [ ] Registration screen should show the selected email.
- [ ] Password field and submit button should be visible.

## Scenario 4.2: Password validation

- On registration screen, enter a password shorter than 8 characters.
- [ ] Password should be rejected.
- Enter a password missing letter, number, or special character.
- [ ] Password should be rejected with rule message.
- Enter a valid password.
- [ ] Password should be accepted for submission.

## Scenario 4.3: Successful registration

- Submit registration with unused email and valid password.
- [ ] Account should be created.
- [ ] Activation link request should be sent.
- [ ] Newly registered user should be logged out.
- [ ] App should route to `/registration-verify`.
- [ ] Verification waiting screen should be shown.

## Scenario 4.4: Already used email

- Start registration with an email that is already registered.
- Submit a valid password.
- [ ] Registration should not complete.
- [ ] Error message should be shown.
- [ ] User should remain in the registration flow.
