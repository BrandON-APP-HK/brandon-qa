# 3. User Login Email Password

## Scope

Email-first login, existing email routing, password login, password errors, and forgot-password entry.

## Preconditions

- Tester is logged out.
- A verified email/password account exists.
- An unverified email/password account exists for verification routing.

## Scenario 3.1: Existing email routes to password screen

- Open the login bottom sheet.
- Tap **Continue with Email**.
- Email login screen should open.
- Enter an invalid email and try to continue.
- Invalid email should be rejected.
- Enter an existing account email.
- Tap **Continue**.
- Password screen should open.
- Selected email should be displayed.
- Password field, **Continue**, and **Forgot password** should be visible.

## Scenario 3.2: Verified email/password login

- Navigate to the password screen for a verified account.
- Enter the correct password.
- Tap **Continue**.
- Login should succeed.
- App should navigate to **Home**.
- User should be logged in.
- Protected tabs should be accessible.

## Scenario 3.3: Incorrect password

- Navigate to the password screen for an existing account.
- Enter an incorrect password.
- Tap **Continue**.
- Login should not succeed.
- Password screen should remain visible.
- Error message should be shown.
- App should not navigate to **Home**.

## Scenario 3.4: Forgot password entry

- Navigate to the password screen.
- Tap **Forgot password**.
- App should route to **Forgot Password**.
- Email should be prefilled where supported.
