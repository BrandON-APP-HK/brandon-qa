# 20. Add Deal Create Submit

## Scope

Add Deal location, store search, required fields, preview, and submit flow.

## Preconditions

- User is logged in for successful submit.
- Location permission can be granted.
- Google Places/API and backend are available.

## Scenario 20.1: Required field validation

- Open Add Deal.
- [ ] Submit button should be disabled until required data is ready.
- Add a photo but do not select store/category.
- [ ] Submit should remain unavailable or show incomplete form recovery.
- Select store but no category.
- [ ] Submit should remain unavailable or show incomplete form recovery.
- Select at least one category.
- [ ] Form should become valid when photo, store, location, and category are ready.

## Scenario 20.2: Location denied/out of range

- Reset location permission.
- Open Add Deal.
- Deny location.
- [ ] Location recovery card should appear.
- [ ] Store search should be blocked.
- Tap Retry when available.
- [ ] App should request location again.
- Test blocked permission.
- [ ] Open Settings action should be available.
- Test out-of-range location if possible.
- [ ] Out-of-range recovery should block store search.

## Scenario 20.3: Store search

- Grant location and open Add Deal.
- Type one or two characters in store search.
- [ ] No search results should appear.
- Type more than two characters.
- [ ] Store search should start after debounce.
- [ ] Results should exclude permanently closed places.
- [ ] Typed manual store option should appear at top.
- Select a result.
- [ ] Picked store section should appear.

## Scenario 20.4: Complete and submit deal

- Add photo.
- Select store.
- Select categories.
- Select promotion type.
- Enter details up to the allowed character limit.
- Toggle unknown end date or select an end date.
- Review final preview.
- Tap submit/add deal.
- [ ] Loading indicator should show while submitting.
- [ ] Deal should be created and photo uploaded.
- [ ] App should navigate to Home with success toast.

## Scenario 20.5: Submit failure and retry

- Complete all required fields.
- Simulate API/network failure.
- Submit.
- [ ] Submit recovery card should appear.
- Tap Retry.
- [ ] Same form data should be submitted again.
- Dismiss recovery.
- [ ] User should remain on Add Deal with entered data preserved.
