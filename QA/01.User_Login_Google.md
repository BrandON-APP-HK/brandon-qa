# 1. User Login Google

## Scope

Google authentication from the shared login bottom sheet.

## Preconditions

- Tester is logged out.
- Google sign-in is configured for the current environment.
- A valid Google account is available on the device.

## Scenario 1.1: Successful Google login from guarded action

- Start the app.
- Home screen should be visible.
- Tap **Add Deal**, **Likes**, or **Me**. (note : what about tapping Map pin icon (top right)?)
- Login bottom sheet should open instead of the protected screen.
- Google login button should be visible.
- Tap **Continue with Google**.
- Native Google sign-in UI should open.
- Complete Google authentication.
- Login bottom sheet should close.
- App should return to **Home**.
- Protected tabs should now be accessible.
- **Me** tab should show the user avatar when available.

## Scenario 1.2: Cancel Google login

- Start as a logged-out user.
- Open the login bottom sheet.
- Tap **Continue with Google**.
- Cancel the native Google flow.
- App should not crash.
- User should remain logged out.
- Protected tabs should still open the login bottom sheet.
- No protected screen should open after cancellation.
