# 14. Map Pin Carousel Redo Search

## Scope

Map pins, active/inactive markers, clusters, carousel sync, and redo search. QR/scanner check-in is excluded.

## Preconditions

- Logged-in user.
- Test data includes regions with fewer than 10 deals and regions with 10 or more deals.
- Map can load deal geo points.

## Scenario 14.1: Individual markers and carousel sync

- Open **Map** in a region with individual pins.
- Bottom carousel should appear when deals are loaded. Displaying selected deal or deal closest to selected center of map.
- Swipe carousel to another deal, 2nd closest to center.
- Active carousel index should change.
- Map should move toward the carousel deal location.
- Selected carousel deal pin should render above nearby or overlapping inactive pins, and change color to be visually identified.
- Tap a visible pin.
- Carousel should scroll to the tapped deal. Map centers on that tapped deal.
- Tapped pin should appear active/larger when selected.
- Tapped pin should render on top when pins share the same or nearby geo.
- Tapped pin should be filled (red for promo block, blue for social deal) to be visually identified easily 

## Scenario 14.2: Active and inactive pin states

- Open map with social deals and promo blocks.
- Inactive markers should use inactive marker styling/assets.
- Active marker should be visually larger than inactive markers.
- Promo block markers should be visually distinguishable from social deal markers.
- Tap between multiple pins.
- Only the selected pin should appear active at a time.
- Selected pin should always render on top, whether selected from the map or carousel.

## Scenario 14.3: Cluster behavior

- Open a region with 10 or more nearby deals.
- Zoom out if needed.
- Numbered cluster markers should appear.
- Cluster marker should show the number of grouped points.
- Promo and social clusters should use their configured colors/styles.
- Tap a cluster.
- Camera should zoom toward the cluster.
- Zoom in beyond cluster threshold.
- Individual pins should replace clusters.

## Scenario 14.4: Redo search

- Open general map.
- Pan to a different area.
- Tap **REDO SEARCH**.
- Search should run around the current map center.
- Search radius should be capped by app behavior and not exceed the configured max radius.
- Active carousel index should reset to the first result when results exist.
- Loading state should replace/disable redo search while the scan is active.
- No-deals message should appear when the search returns no results.

## Scenario 14.5: Source-backed supplied map rules

- Open map from a selected social deal/card.
- Map should focus the selected active deal pin.
- Search region should be set around the selected deal.
- Tap a small pin/card marker in the carousel.
- Map should focus that active pin.
- Tap redo search after panning.
- Map should focus the center of the searched area and update carousel results.
