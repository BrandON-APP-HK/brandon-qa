# 7. Navigation Guest Access Control

## Scope

Guest restrictions for bottom tabs, drawer items, and auth-required actions.

## Preconditions

- Tester is logged out.

## Scenario 7.1: Guest public access

- Start the app.
- `/home` should be visible.
- Open drawer.
- Tap **Help Center**.
- Help Center should open without login.
- Return and tap **About**.
- Learn About should open without login.

## Scenario 7.2: Guarded tabs open login

- As guest, tap **Search**.
- Login bottom sheet should open instead of Search.
- Dismiss it.
- Tap **Add Deal**.
- Login bottom sheet should open instead of Add Deal.
- Dismiss it.
- Tap **Likes**.
- Login bottom sheet should open and liked-only should not toggle.
- Dismiss it.
- Tap **Me**.
- Login bottom sheet should open instead of Account.

## Scenario 7.3: Auth-only drawer items hidden for guest

- Open drawer while logged out.
- **My Social Activity** should be hidden.
- **Following Brands** should be hidden.
- **My Points** should be hidden.
- **Help Center** should be visible.
- **About** should be visible.
- Drawer footer should show **Login**.

## Scenario 7.4: Protected navigation after login

- Log in successfully.
- Tap **Search**, **Add Deal**, **Likes**, and **Me**.
- Protected tabs should no longer show login bottom sheet.
- **Likes** should toggle liked-only state.
- **Me** should open profile.
- Open drawer.
- Auth-only drawer items should be visible.
