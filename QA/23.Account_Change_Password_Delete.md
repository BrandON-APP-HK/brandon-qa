# 23. Account Change Password Delete

## Scope

Change password and delete account modal flows.

## Preconditions

- User is logged in.
- Password-change scenarios use password-login account.

## Scenario 23.1: Change password validation

- Open Account > Change Password.
- Tap OK with empty fields.
- Required errors should show.
- Enter mismatched new and repeat passwords.
- Mismatch error should show.
- Enter password shorter than 8 characters or missing required pattern.
- Password rule errors should show.

## Scenario 23.2: Change password success and errors

- Enter correct old password and valid matching new password.
- Tap OK.
- Modal should close or navigate back.
- Success toast should show.
- Reopen and enter wrong old password.
- Invalid password error should show.
- Trigger too many attempts if possible.
- Too-many-requests error should show.

## Scenario 23.3: Delete account cancel

- Open Account preferences.
- Tap delete/close account.
- Delete account confirmation modal should open.
- Tap Cancel.
- Modal should close and user should remain logged in.

## Scenario 23.4: Delete account confirm

- Open delete account modal.
- Tap OK.
- Delete request should start.
- On success, user should be signed out and dismissed to Home.
- On recent-login/error failure, Account should show error toast asking user to log in again.
- Protected tabs should require login after successful deletion.

- Note : for accounts deleted, any ongoing social deal posted from that account remains available on the app, but is anonymized (default user profile, "anonymous" name).
